算法的时间复杂度

算法的时间复杂度指的是算法在终止之前需要多少时间。
由于更庞大的输入往往会导致更长的运行时间，因此算法的时间复杂度应当是一个关于输入大小的函数，比如T(n)。
对于某个算法的时间需求，在不同情况下（例如最坏情况、最佳情况和平均情况）会需要不同的时间，我们主要关注最坏情况下需要的时间。无论输入有多大，算法的运行时间总不会超过最坏情况。
（注：尽管有的算法在最坏情况下需要很长时间，但是在实践中往往不会遇到最坏情况，因此我们还是会使用这些算法）
对于一个算法以及一个相应的输入，我们采取RAM模型对其所需的时间进行计算：

  • 每个基本操作都消耗常数时间
  • 每个内存访问都消耗常数时间
  • 每个复杂操作都应被分解为基本操作
  • 最后计算这些时间单位的总和

渐进分析

然而，在实践中我们往往不会关心计算算法复杂度时的一些不重要的细节。
因此我们需要更简洁的抽象，让我们能够对某个算法的复杂度有一个更直观的感受。
这层抽象指的是算法运行时间的增长速率。在这层抽象中，两个因素被忽略了：

  • 常数系数
  • 低阶项

（注：这些被忽略的因素都基于一个较大的n,也就是输入大小）

  1. ‘O’表示法
    对于一个函数g(n)，O(g(n)) = {f(n) ∣ ∃c > 0,∃n0 > 0,∀n ≥ n0 : 0 ≤ f(n) ≤ c ⋅ g(n)}
  2. ‘Ω’表示法
    对于一个函数g(n)，Ω(g(n)) = {f(n) ∣ ∃c > 0,∃n0 > 0,∀n ≥ n0 : f(n) ≥ c ⋅ g(n)}
  3. ‘Θ’表示法
    对于一个函数g(n)，Θ(g(n)) = {f(n) ∣ ∃c1 > 0,∃c2 > 0,∃n0 > 0,∀n ≥ n0 : c1 ⋅ g(n) ≤ f(n) ≤ c2 ⋅ g(n)}
  4. ‘o’表示法
    对于一个函数g(n)，o(g(n)) = {f(n) ∣ ∀c > 0,∃n0 > 0,∀n ≥ n0 : 0 ≤ f(n) < c ⋅ g(n)}
  5. ‘ω’表示法
    对于一个函数g(n)，ω(g(n)) = {f(n) ∣ ∀c > 0,∃n0 > 0,∀n ≥ n0 : f(n) > c ⋅ g(n)}
